Just starting a new project. Please create a PDD for a low-cost, dual-mode RTK GNSS system using an ESP32 with esp-idf and a u-blox ZED-F9P that can operate as either a rover or a base station, with support for both internet-based (NTRIP) and long-range direct communication.  In rover mode, the device should receive RTCM correction data via either an NTRIP client over WiFi or a local communication link (e.g., radio or direct WiFi), pass corrections to the GNSS module, and output real-time centimeter-accuracy positioning. It should include a button-based capture workflow that records points only when RTK FIX is stable, including full metadata for GIS use.  In base mode, the device should configure the GNSS module for survey-in or fixed-position operation, generate RTCM correction messages, and transmit them via a pluggable transport layer (WiFi, NTRIP caster, or long-range radio) for use by rovers.  The system must separate GNSS processing, communication (transport layer), and application logic, allowing interchangeable communication methods without modifying core positioning logic.  Hardware should be battery-powered, compact, and housed in a 3D-printed enclosure, with an external L1/L2 antenna mounted for optimal sky visibility. The system should support field deployment, long-range communication, and integration with GIS workflows such as ArcGIS (including projected coordinate outputs like State Plane).  

Provide system architecture, wiring, ESP32 firmware design, GNSS configuration for base/rover modes, transport layer design, RTCM data handling, logging format, and best practices for achieving reliable centimeter-level accuracy in real-world environments.
